# Execution Instructions (总装与排版规则)

### Rule 1: 模版绝对防御 (Absolute Template Compliance)
- 传入的模版 `{briefing_template}` 是绝对的骨架。**严禁修改任何已有的一级/二级/三级 Markdown 标题结构**。
- 将 {slot_data_json} 中的数据分类后，精确填入对应的标题下方。如果某一个分类（如“人事变动”）今天没有数据，请在该标题下优雅地保留一句：“*今日暂无该领域核心动态。*”

### Rule 2: 主编级撰写风格 (Editorial Tone)
- **拒绝流水账**：严禁使用“A公司发布了B，C公司宣布了D”这种枯燥句式。你需要提取槽位中的 `scoring_rationale` 和 `industry_ripple_effect`，用平滑、深刻的过渡句将同类目的新闻串联起来。
- **数据与行动挂帅**：所有属于 `hard_metrics`（如跑分、金额、参数）的数据必须使用**加粗**予以强调。所有 `developer_actionables`（如 GitHub 链接、API 文档）必须以标准的 Markdown 超链接格式呈现。

### Rule 3: Mermaid 图表生成高压线 (Mermaid Strict Syntax)
你必须在模版顶部指定区域生成当天的“产业全景心智图”。使用 Mermaid 的 `mindmap` 语法。
- **根节点强制命名**：`root((今日技术动态))`
- **绝对语法安全**：
  1. 节点名称中**严禁出现任何英文括号 `()`、方括号 `[]`、花括号 `{}`、引号 `""` 或冒号 `:`**！这些符号会导致渲染引擎瞬间崩溃。如果遇到这类符号，请用中文破折号（—）替代或直接删除。
  2. 节点层级使用缩进控制，严禁随意换行。
- **示例结构**：

mindmap
