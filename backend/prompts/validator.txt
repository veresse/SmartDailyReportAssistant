# Validation Protocols (强制校验协议)
请依次执行以下 4 项协议的严格校验。只要触发任意一条红线，`is_valid` 必须判定为 `false`！

### 🛑 Protocol 1: Mermaid 渲染安全校验 (最高优)
扫描文档中的 ```mermaid 代码块：
1. **闭合检查**：`mermaid` 代码块必须有首尾完整的 
``` 闭合。
2. **根节点校验**：必须是以 `mindmap` 开头，且根节点必须严格为 `root((今日技术动态))`。
3. **致命字符扫描**：检查内部所有子节点文本。**节点文本中绝对不允许包含以下字符：英文圆括号 `()`、方括号 `[]`、花括号 `{}`、英文双引号 `""`、冒号 `:`。** (例如：`Llama-3(8B)` 将导致渲染引擎彻底崩溃！必须要求其改为 `Llama-3-8B`)。

### 🛑 Protocol 2: 硬核指标防篡改校验 (Data Fidelity)
遍历 {slot_data_json} 中所有条目的 `hard_metrics` 数组：
1. 原数据中出现的具体数字（金额、参数量、跑分、日期），必须在 {filled_markdown} 中**一字不差**地出现。
2. 严禁漏掉任何一个 metric，严禁大模型私自四舍五入或概括（如原数据是 60.5亿，文档写 60亿即为违规）。

### 🛑 Protocol 3: 开发者链接可用性校验 (Link Integrity)
遍历 {slot_data_json} 中所有 `developer_actionables` 包含的链接：
1. Markdown 超链接语法必须完美闭合：`[文本描述](https://...)`。严禁出现 `[文本](链接` 或缺少括号的情况。
2. 链接本身不能在排版时被意外截断。

### 🛑 Protocol 4: 结构完整性校验 (Completeness)
1. {slot_data_json} 中包含的 event_category，其对应的新闻必须出现在文档的对应版块下。
2. Markdown 标题层级 (`#`, `##`, `###`) 必须规范，不能出现错乱或未渲染的标题（如 `##标题` 缺少空格）。
