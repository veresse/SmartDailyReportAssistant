# 📐 V0.3 版本需求文档：个人旗舰版（显式个性化与 3 层去重管线）

> **升级目标**：全面抛弃多用户冗余设计，确立“单用户显式 Prompt 调教”核心架构。通过重构数据入库链路与引入 3 层混合去重机制，打造零打扰、高精准的本地 AI 资讯助理。

## 1. 架构精简与废弃声明 (决议项)

为保障 Token 利用率和处理效率，对 V0.2 现有模块进行以下精简裁撤：

* **废弃 `filter.py` 模块**：由于 V0.3 在 Loop A 的打分阶段已引入强烈的个人偏好判定（评分 `<60` 即为不符合口味或水文），过滤动作已**内聚**于 `scorer.py` 之中。原 Loop B 中的独立大模型过滤步骤将被彻底移除，以避免双重过滤造成的 Token 浪费。
* **废弃相关配置项**：同步移除原有的 `AI_FILTER_ENABLED`、`AI_FILTER_TARGET_AUDIENCE` 和 `AI_FILTER_BATCH_SIZE` 配置。

---

## 2. 核心管线：3 层混合去重机制

系统将确立界限分明、互不干扰的 3 层去重机制，彻底杜绝重复推送。

### 2.1 机制图解

```text
[抓取源数据] 
    ↓
【第 1 层：Loop A 硬拦截】 → 过滤极度相似的跨源重复发文 (阻挡入库)
    ↓
[入库 raw_news_items]
    ↓
【第 2 层：Loop B 语义去重】 → 聚合同一天内发生的相似事件 (合并摘要)
    ↓
【第 3 层：Loop B 脉络去重】 → 过滤与昨天/前天完全重复的旧闻 (阻挡上报)
    ↓
[生成最终晨报 daily_briefings]

```

### 2.2 各层职责与执行细节

* **第 1 层：Loop A 标题硬拦截 (新增)**
* **职责**：防止各大媒体短时间内对同一事件采用相似标题刷屏。
* **机制**：计算新抓取标题与数据库近 24 小时内记录的文本编辑距离（建议引入 `rapidfuzz` 库）。相似度 `> 0.85` 时直接丢弃，不调用 LLM，不入库。


* **第 2 层：Loop B 同日语义去重 (保留现有 `deduplicator.py`)**
* **职责**：处理同一天内，标题完全不同但实质报道同一事件的新闻。
* **机制**：维持 V0.2 逻辑不变，聚合并选出信息最丰富的一条。


* **第 3 层：Loop B 跨天脉络过滤 (新增)**
* **职责**：处理跨日期的重复推送（如某站昨天首发，另一站今天才跟进）。
* **机制**：在 `summarizer.py` 中注入历史摘要，若 LLM 判定为旧事重提，返回 `"category": "[重复已阅]"`，系统在生成最终结构前将此类条目剔除。

## 3. 数据模型与入库规范调整

在不动摇原有结构的前提下，补充标签提取功能的落地细节。

### 3.1 Pydantic Schema 扩展 (`base.py`)

在标准输出类中新增 `ai_tags` 字段：

```python
class RawNewsItemSchema(BaseModel):
    # ... 保留原有字段 ...
    ai_tags: list[str] = Field(default_factory=list) # 新增：AI提取的实体标签

```

### 3.2 数据库字段定义 (`models.py`)

在 `RawNewsItem` 模型中增加对应列：

```python
ai_tags = Column(Text, default="[]") # 存储 JSON 序列化后的字符串

```

### 3.3 数据入库序列化规范 (`jobs.py`)

在 Loop A 获取到 `RawNewsItemSchema` 并向数据库执行写入时，必须使用 `json.dumps(item.ai_tags, ensure_ascii=False)` 将列表转换为 JSON 字符串存入 `ai_tags` 列。

## 5. 跨天历史脉络注入 (Loop B 改造)

在 `summarizer.py` 中重写 Prompt，赋予 LLM 短期记忆。

### 5.1 Token 预算控制 (Soft Limit)

为防止历史数据过多撑爆大模型上下文窗口，执行严格的条数限制：

* 查询过去 `CONTEXT_LOOKBACK_DAYS`（默认 3）天的数据。
* 强制截断：最多仅注入 **前 45 条** 历史摘要（按时间倒序或评分降序），确保历史上下文文本量控制在 1500 Tokens 以内。

### 5.2 上下文 Prompt 模板

```python
_CONTEXTUAL_SUMMARY_PROMPT = """你是一个资深的 AI 行业分析师。

【今日新闻】
- 标题：{current_title}
- 内容：{current_description}

【🧠 短期记忆：近期已阅资讯】
{historical_context}

【任务要求】
1. 去重判断：如果今日新闻与【短期记忆】中某件事完全是重复报道，请务必在 `category` 返回 `[重复已阅]`。
2. 脉络追踪：如果是【短期记忆】中某件事的后续进展，请在摘要中明确指出。

请返回 JSON：
{{
    "one_line_summary": "一句话概括",
    "key_points": ["要点1", "要点2", "结合历史脉络的分析"],
    "importance": "为什么重要",
    "category": "标签"
}}
"""

```

## 6. 定时清理任务挂载修正

更正字段引用错误，明确任务调度器的挂载位置。

### 6.1 清理逻辑校准 (`jobs.py`)

方法 `cleanup_memory()` 中的过滤条件必须使用 **`collected_at`** 字段：
执行物理删除：`DELETE FROM raw_news_items WHERE collected_at < (当前时间 - CLEANUP_RETENTION_DAYS)`。

### 6.2 调度器注册 (`main.py`)

必须在 FastAPI 的 `lifespan` 阶段显式挂载清理任务，与其他 Loop 并行：

```python
scheduler.add_job(
    cleanup_memory,
    CronTrigger(hour=settings.cleanup_hour, minute=settings.cleanup_minute, timezone=settings.timezone),
    id="cleanup_old_data",
    replace_existing=True
)

```

## 7. 全局配置项补充 (`.env`)

在 `config.py` 和 `.env.example` 中补充以下必须的系统级环境变量：

| 变量名 | 类型 | 默认值 | 作用说明 |
| --- | --- | --- | --- |
| `USER_PERSONA` | `str` | `关注AI开发的程序员...` | Loop A 决定新闻生杀大权的私人画像 |
| `TITLE_SIMILARITY_THRESHOLD` | `float` | `0.85` | Loop A 标题相似度硬拦截阈值 |
| `CONTEXT_LOOKBACK_DAYS` | `int` | `3` | Loop B 向前追溯历史脉络的天数 |
| `CONTEXT_MAX_ITEMS` | `int` | `45` | Loop B 注入历史上下文的最大条数（防 Token 溢出） |
| `CLEANUP_RETENTION_DAYS` | `int` | `7` | 数据库垃圾数据保留天数 |
| `CLEANUP_HOUR` | `int` | `3` | 数据清理任务执行时间（小时） |
| `CLEANUP_MINUTE` | `int` | `0` | 数据清理任务执行时间（分钟） |
