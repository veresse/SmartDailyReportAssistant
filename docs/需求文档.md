# 📝 产品需求文档 (PRD)：智能 AI 开发者早报系统 (AI Morning Briefing)

## 1. 产品概述 (Product Overview)

**产品定位**：一款专为开发者、研究员及科技爱好者打造的自动化、个性化 AI 新闻雷达与分发平台。
**核心价值**：解决信息过载问题，通过多源聚合、AI 深度处理（去重、背景补充）、个性化偏好学习，以及创新的“知识图谱可视化”推送，将高质量的技术情报精准触达至用户的企业协作工具（飞书/钉钉）及前端看板。

## 2. 核心系统架构 (System Architecture)

系统主要分为五个层级：**数据采集层** ➡️ **AI 处理中枢** ➡️ **个性化推荐引擎** ➡️ **分发推送层** ➡️ **前端交互层**。

---

## 3. 详细功能模块需求 (Detailed Requirements)

### 3.1 数据采集层 (Data Collection)

* **多源异构数据接入**：
* **GitHub**：拉取 Trending 榜单、特定领域的优质 Release 动态。（示例代码在test\github_trending.py）
* **Hacker News**：拉取 Top Stories，按 Score 过滤，并抓取高价值评论。（示例代码在test\HackerNews_front.py）
* **Hugging Face**：获取每日 Trending Models, Papers 和 Spaces。（示例代码在test\huggingfacedailypaper.py）


* **定时调度机制**：支持 Cron 表达式配置抓取频率（如每天凌晨 6 点执行抓取任务）。

### 3.2 AI 处理中枢 (AI Processing Hub)

这是系统的“大脑”，依托大模型（如 GPT-4, Claude 3.5 或 DeepSeek）进行深度加工。

* **多源消息去重 (Deduplication)**：将指向同一事件的不同报道（如 HN 上的帖子和 GitHub 的同一个项目）进行语义级别的合并，提取核心信息，避免重复阅读。
* **背景知识补充 (Contextual Enrichment)**：识别新闻中的专业术语、缩写或新出现的公司/项目，调用 AI 或结合联网搜索（如 Google Search API）补充背景释义。
* **结构化摘要 (Structured Summarization)**：将长文转化为“一句话结论 + 3个核心要点 + 为什么重要”的标准结构。
* **🌐 核心创新：技术演进思维导图 (Mind Map Generation)**：
* **逻辑聚类**：AI 自动分析当天所有新闻，提取共性主题（如“开源大模型动态”、“前端框架更新”）。
* **Mermaid 代码生成**：利用 LLM 针对这些主题输出标准的 Mermaid 代码（如 `graph TD` 或 `mindmap`）。
* **Markdown 渲染**：将 Mermaid 代码无缝嵌入到推送到飞书/钉钉的 Markdown 文本中，确保用户端直接看到直观的关联图谱。



### 3.3 个性化与反馈反馈层 (Personalization & Feedback Loop)

让系统“越用越懂你”，实现千人千面的推送。

* **隐式偏好收集**：
* **前端埋点**：记录用户在前端面板的浏览行为（如：在 Hugging Face 分类下的停留时长、点击原文链接的次数）。


* **显式偏好收集**：
* 在前端界面或飞书/钉钉卡片底部提供 👍（增加此类）/ 👎（减少此类）按钮。


* **动态 Prompt 调整 (Dynamic Prompting)**：
* 构建用户画像数据库（User Profile）。
* 在生成第二天的日报前，系统会将用户的画像（例如：“该用户对 Hugging Face 开源模型高度关注，对前端新闻不感兴趣”）注入到 AI 总结和过滤的 Prompt 中，自动提高此类新闻的权重，并增加相关背景的深度。



### 3.4 消息推送层 (Push Notification)

* **平台适配**：支持配置钉钉群机器人、飞书自定义机器人（Webhook）。
* **富文本卡片排版**：
* 使用飞书/钉钉的卡片消息（Message Card）格式。
* **顶部**：当天头条概览与“技术演进思维导图”（Mermaid 渲染）。
* **主体**：按优先级排序的 3-5 条核心新闻（带分类 Tag）。
* **底部**：互动反馈按钮（如有）。



### 3.5 前端交互层 (Frontend Interface)

* **历史早报看板**：日历视图，方便查阅过往所有早报。
* **沉浸式阅读模式**：提供更丰富的交互体验，支持点击新闻查看 AI 补充的详细背景和长文翻译。
* **个人偏好设置台**：用户可在此可视化地调整自己的兴趣权重（拖拽滑块调整 GitHub、HN 的比重），查看自己的“阅读偏好画像”。

## 4. 业务流转过程 (Workflow)

1. **06:00** 抓取脚本启动，从各大平台拉取过去 24 小时的原始数据。
2. **06:15** 数据进入 AI 中枢，执行去重、打分、翻译、背景补充。
3. **06:30** 读取用户/群组的偏好画像，AI 筛选出最符合该受众的 Top 10 新闻。
4. **06:40** AI 针对这 Top 10 新闻提取主题，生成 Mermaid 思维导图代码。
5. **07:00** 组装富文本卡片，触发 Webhook，推送到飞书/钉钉；同时更新前端数据库。
6. **08:00+** 用户阅读新闻，前端和机器人记录点击与停留时长，更新用户画像，为明天的推送做准备。

## 5. 关键技术选型建议 (Tech Stack Recommendation)

* **后端语言**：Python (最适合数据抓取与 AI 接口对接)。
* **爬虫框架**：`Scrapy` 或轻量级的 `BeautifulSoup` + `requests`（针对提供 API 的平台直接调 API）。
* **大模型调用**：LangChain 或直接使用 OpenAI / DeepSeek / 智谱等 API。
* **前端框架**：Vue 3 或 React + Tailwind CSS。
* **数据库**：本地SQLite (存储新闻与配置) + Redis (缓存与前端埋点高频写入)。
* **调度系统**：Celery 配合 Redis，或轻量级的 APScheduler。
